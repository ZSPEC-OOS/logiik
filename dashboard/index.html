Welcome to NERO
NERO is an AI model that learns by studying questions and answers provided by a powerful teacher AI (Kimi K2.5). This guide walks you through everything — no coding knowledge required.
What is NERO? Before you start Running the server Step 1 · Setup Step 2 · Training Step 3 · Monitor Step 4 · Test AI Exporting Glossary FAQ

What is NERO?

NERO is a student AI that gets smarter by learning from a teacher AI. The teacher (Kimi K2.5) generates training questions and multiple possible answers. NERO studies all of those question–answer pairs and gradually develops the ability to answer new questions on its own — without looking anything up or copying from the training data.

Training happens in three phases, each building on the last:

Phase 1 — Memorization

NERO studies factual Q&A pairs. It learns the basic structure of questions and answers.

Phase 2 — Generation

NERO starts forming its own answers rather than just selecting from given options.

Phase 3 — Abstraction

NERO connects ideas across different topics and produces more sophisticated responses.

Before You Start

You will need three things from your Kimi K2.5 account:

1
API Key — a secret password that lets NERO talk to Kimi K2.5. It usually looks like sk-abc123…. Keep it private — do not share it.
2
Base URL — the web address of the Kimi K2.5 API endpoint. Example: https://api.moonshot.cn/v1
3
Model ID — the exact name of the model to use. Example: kimi-k2-5. Check your Kimi dashboard for the correct ID.
Tip: You also need to decide what topics you want NERO to learn about. You can use plain English phrases like history of the Roman Empire, basic mathematics, or creative writing. NERO will ask Kimi K2.5 to generate training questions on those subjects.

Running the NERO Server

The dashboard you are reading right now is served by the NERO backend. If you can see this page, the server is already running and you can skip this section.

If you need to start it yourself, open a terminal in the NERO project folder and run:

uvicorn cognita.api.server:app --host 0.0.0.0 --port 8000

Then open your browser and go to http://localhost:8000.

Note: The server must stay running the whole time you are training. If you close the terminal window, training will stop.

Step 1 — Setup & Initialize

Click the ⚙ Setup tab (or the Setup & Initialize button in the sidebar).

1
Teacher API Key — paste your Kimi K2.5 API key here. The field hides what you type for security.
2
Teacher Base URL — paste the API base address, e.g. https://api.moonshot.cn/v1.
3
Model ID — type the model name, e.g. kimi-k2-5.
4
Training Topics — type the subjects you want NERO to learn, one per line. Examples:
basic_reasoning fact_recall creative_writing science_concepts
5
Total Examples — how many question–answer pairs the teacher should generate in total. Start with 100 for a quick test. Higher numbers mean more learning but take longer.
6
Leave Brain ID, Knowledge Base Path, and Firebase at their defaults unless you know what they do.
7
Click ⚡ Initialize System. You will see a green confirmation message and the sidebar will change to Initialized. The Start Training button will become clickable.
Already trained before? NERO automatically loads your last saved checkpoint when you initialize. You will see how many checkpoints were found in the confirmation message.

Step 2 — Start Training

Once initialized, click ▶ Start Training in the sidebar. The status badge in the bottom-left will turn green and start pulsing.

What happens behind the scenes:

  1. NERO asks Kimi K2.5 to generate batches of question–answer pairs on your chosen topics.
  2. For every question, Kimi produces 5–10 possible answers of varying quality.
  3. NERO studies each question paired with each answer independently, learning the full range of how a question can be answered.
  4. Every 50 training steps NERO tests itself on a held-out set of examples it has never seen, to check it is genuinely learning and not just memorising.
  5. When progress on the held-out test plateaus, NERO automatically advances to the next curriculum phase.
  6. A checkpoint (saved snapshot) is written every 100 steps.

Click ⏹ Stop Training at any time to pause. Your progress is saved automatically.

Heads up: Training makes real API calls to Kimi K2.5 and will consume your API credits. Start with a small number of Total Examples (e.g. 50–100) to test your setup before running a long session.

Step 3 — Monitoring Progress

Click the 📈 Monitor tab to watch training in real time.

Curriculum Phase cards

Three cards show which phase NERO is in. The active phase is highlighted and labelled Active. Completed phases are labelled Done. Upcoming ones are faded and labelled Pending.

Loss Curves chart

This chart has two lines:

  • Train Loss (red) — how wrong NERO is on the examples it is currently studying. You want this to go down over time.
  • Val Loss (teal dashed) — how wrong NERO is on a separate set of examples it has never trained on. This is the most honest measure of learning. You want this to go down too.
What if Train Loss drops but Val Loss stays high? This is called overfitting — NERO has memorised the training examples instead of truly understanding them. Try reducing Total Examples or adding more variety to your topics.

Validation Perplexity chart

Perplexity is a measure of how surprised NERO is by correct answers. A lower number is better. Think of it like this:

PerplexityWhat it means
< 10Excellent — NERO is very confident on these topics
10 – 50Good — solid learning, room to improve
50 – 200Fair — still learning, early stages
> 200Poor — training has just started or something may be wrong

Training Log

A live terminal-style panel at the bottom of the Monitor tab. It records every validation result, phase change, connection event, and error in real time. Use the Clear button to wipe it. It keeps the last 200 entries.

Step 4 — Testing NERO

Click the 💬 Test AI tab to ask NERO questions directly. NERO must be initialized first; it does not need to be actively training.

1
Type your question in the text box.
2
Leave Require original answer checked. This tells NERO to only return a response it is confident it generated itself rather than recalled from memory. Uncheck it if you want responses even when NERO is less certain.
3
Click Generate Response and wait a few seconds.

Below the answer you will see three indicators:

IndicatorWhat it means
ConfidenceHow sure NERO is about its answer (higher % is more confident)
OriginalYes means NERO generated this rather than copying; No means it was less confident
TokensApproximate length of the answer in word-pieces (roughly 1 token ≈ ¾ of a word)
Early in training? Responses will be rough or incoherent — that is normal. NERO needs hundreds or thousands of training steps before producing fluent answers. Keep training and test again later.

Exporting Your Knowledge Base

Click 📥 Export Package in the sidebar to save a portable copy of everything NERO has learned to your server's disk. The export path is shown in the Training Log.

The export includes the LoRA adapter weights, the generative head, and all metadata. You can copy this folder to another machine and load it the next time you initialize by pointing the Knowledge Base Path field at it.

The 🗄 Knowledge tab shows you what is currently stored: how many checkpoints exist, how many embeddings have been saved, and the details of the most recent checkpoint including its loss and perplexity.

Glossary

Loss
A number measuring how wrong the model's predictions are. Lower is better. Zero would mean perfect.
Validation Loss
Loss measured on examples the model has never trained on — a more honest measure of learning than training loss.
Perplexity
How surprised the model is by correct answers. Lower is better. It is calculated as e raised to the power of the validation loss.
Overfitting
When a model memorises training examples instead of learning general patterns. Signs: training loss keeps falling but validation loss stops improving or rises.
Checkpoint
A snapshot of NERO's weights saved to disk at regular intervals so training can resume or be rolled back.
LoRA
A technique that fine-tunes only a small fraction of the model's parameters, making training faster and cheaper without sacrificing quality.
Optimizer Step
One cycle of adjusting NERO's internal weights based on what it got wrong. Gradients are accumulated over 4 batches before each step.
Curriculum Phase
A stage of training with a different goal. NERO moves through Memorization → Generation → Abstraction automatically when its validation loss stops improving.
Teacher AI
Kimi K2.5, the external AI that generates the training questions and answers NERO learns from.
Knowledge Distillation
A technique where the student model (NERO) learns not just the right answer but also the teacher's probability distribution over all possible answers — a richer signal than a single label.
Token
A chunk of text roughly equal to ¾ of a word. Language models read and write in tokens, not characters or words.

Frequently Asked Questions

The status says "Not initialized" — what do I do?
Go to the Setup tab, fill in your Kimi K2.5 credentials and topics, then click Initialize System.
Start Training is greyed out.
You must initialize the system first. Complete the Setup tab and click Initialize System.
The Training Log shows "WebSocket disconnected".
The dashboard lost its live connection to the server. It will reconnect automatically in 3 seconds. If it keeps happening, check that the NERO server is still running in your terminal.
My validation loss is going up while training loss goes down.
This is overfitting. NERO is memorising instead of learning. Try adding more diverse topics, reducing Total Examples per batch, or stopping training and testing with the current checkpoint.
How long does training take?
It depends on your hardware and how many examples you requested. On a modern GPU, 100 examples typically take a few minutes. On CPU it will be significantly slower. Watch the Training Log and Optimizer Step counter to track progress.
Can I stop and resume training?
Yes. Click Stop Training — NERO saves a checkpoint automatically. Next time you initialize (with the same Knowledge Base Path), the latest checkpoint is loaded and you can click Start Training to continue.
NERO's answers look like nonsense.
This is normal early in training. The model needs many optimizer steps before generating coherent text. Keep training, watch the perplexity chart drop, and test again after a few hundred steps.
I accidentally closed the browser. Is my training lost?
No. Training runs on the server, not in the browser. Close and reopen the page freely — the server keeps running and the WebSocket reconnects automatically.
What is the Firebase field for?
Firebase is an optional cloud storage service. If you leave it blank, all checkpoints and embeddings are saved locally on the server. You only need Firebase if you want to sync NERO's memory across multiple machines.
