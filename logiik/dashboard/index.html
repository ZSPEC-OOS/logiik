How To Build Your Training Dataset

STEP 1 — Switch to local mode (no accounts needed)

Open logiik/config/config.yaml and change one line:

backend: "faiss"   # was "pinecone" — stores vectors locally

You don't need Pinecone or Firebase yet. FAISS saves everything to disk for free.

STEP 2 — Run generation (CPU, no GPU needed)
python -m logiik.main

Watch this dashboard. Each phase finishes when Coverage hits 95% and Saturation hits 90%. All 12 phases must complete before exporting.

STEP 3 — Export to training files
python -m logiik.utils.export_dataset --output_dir ./training_data

Creates one JSONL file per phase + all_phases.jsonl. Check export_summary.json to verify example counts.

STEP 4 — GPU training: Phases 1–11 (SFT)
pip install trl transformers datasets accelerate bitsandbytes peft

python -m logiik.core.sft_trainer \
  --data_dir   ./training_data \
  --model_name mistralai/Mistral-7B-v0.1 \
  --output_dir ./checkpoints

Uses QLoRA 4-bit by default — works on a single 24GB GPU. Swap the model name for any HuggingFace causal LM.

STEP 5 — Phase 12 RL fine-tuning (PPO)
python -m logiik.core.sft_trainer \
  --data_dir   ./training_data \
  --model_name ./checkpoints/final \
  --output_dir ./checkpoints_phase12 \
  --phases 12 --no_4bit

Refines judgment and calibration on top of the SFT base using PPO/TRL. Run this after Step 4 completes.

What you need right now
✓ Generation — any CPU machine
✓ FAISS — built in, zero setup
✓ Firebase — optional, skip for now
✓ Pinecone — optional, skip for now
